# Public Dashboard — Full View Design

**Date:** 2026-04-18
**Status:** Approved

## Overview

Redesign the public dashboard at `/view/[token]` to show the full dashboard content — including province filters, area toggle, all stat cards, and all charts — while hiding admin-only controls and ensuring no raw data is exposed to unauthenticated viewers.

---

## Architecture & Data Flow

### Shared stats extraction

Move `getStats(provinceFilter)` from `app/page.tsx` into `lib/dashboard-stats.ts`. Extend its signature to accept `areaMode` as well so both callers can pass it cleanly. `app/page.tsx` swaps its local definition for an import. The public page imports the same function.

### Public page URL params

```
/view/[token]?provinces=CAMARINES+SUR,ALBAY&area=original
```

- `provinces` — comma-separated province names, same format as the private dashboard
- `area` — `validated` (default) or `original`

Both params are read server-side on each request; no client-side state is needed.

### Province list for the filter UI

The public page fetches `allProvinces` with a single `groupBy` on `province_edited` — province names only, no sensitive fields.

## Components & Layout

Rendered top to bottom inside a standalone `min-h-screen bg-gray-50` shell (`max-w-6xl mx-auto`). No shared authenticated layout, no sidebar.

| # | Component | Notes |
|---|-----------|-------|
| 1 | **Header** | DAR logo, title, timestamp, Live badge — unchanged |
| 2 | **Area Toggle** | `DashboardAreaToggle` driven by `?area=` param |
| 3 | **Province Filter** | `DashboardProvinceFilter` driven by `?provinces=` param |
| 4 | **Stat Cards** | `DashboardStatCards` with full props from shared `getStats()` |
| 5 | **Charts Row 1** | Records by Province + Records by Status |
| 6 | **Charts Row 2** | COCROM Encoding + COCROM Distribution |
| 7 | **Not Eligible for Encoding** | Full-width chart (by province + by reason) |
| 8 | **Footer** | "Read-only public summary. Data is live and updated in real time." |

**Explicitly hidden:**
- Sidebar (absent from public layout)
- Share button
- Export buttons
- IssueStrip (data quality flags — internal only)
- DashboardProgress / Accomplishment Tracker
- Sign Out / user info

## Security

### Access control
- Token validated server-side before any DB query runs — invalid token → immediate `notFound()`
- Token is a 32-character hex string (128-bit entropy), not guessable
- Token can be regenerated by super_admin at any time; old URL immediately returns 404

### Data exposure
- All Prisma queries run server-side inside React Server Components
- The two `findMany` calls (COCROM charts) use narrow `select` clauses and are fully aggregated before the page renders — only computed numbers (counts, sums) reach the browser
- No raw Landholding or ARB records, landowner names, or seqno_darro identifiers are sent to the client
- No client-fetchable API endpoint is exposed for the public view — all data is SSR

### Inaccessible from public view
- `/api/admin/*` routes
- `/records`, `/users`, `/audit`, `/arbs`, `/batch` pages
- Session tokens, user info, role data

## Implementation Steps

1. Extract `getStats()` from `app/page.tsx` → `lib/dashboard-stats.ts`; update import in `app/page.tsx`
2. Update `app/view/[token]/page.tsx`:
   - Read `?provinces=` and `?area=` search params
   - Fetch `allProvinces` for the filter UI
   - Call shared `getStats()` with province + area params
   - Build all derived data (cocromEncodingData, cocromDistributionData, notEligible charts, etc.)
   - Render full component layout as specified above
3. Remove placeholder empty arrays (`emptyEncoding`, `emptySourceRows`, etc.)
4. Verify no raw records reach the client (audit `select` clauses on `findMany` calls)
